# Tasks

## Task Format

- 本ドキュメントは「実装単位（Work Unit）」ごとに管理する。
- 各Work Unitは以下を持つ。
  - 背景
  - スコープ
  - サブタスク（チェックリスト）
  - 完了条件
- `docs/design.md` は最新目標のみを持ち、変更の背景・移行説明はここに記載する。

## Active Work Units

### [PHYS-04] 地形更新のSub-block起動（Chunk正本維持）

- Status: `Planned`
- 背景:
  - 地形の破壊/分離判定を `Chunk` 粒度だけで起動すると単位が粗く、局所更新の利点が薄れる。
  - 一方、保存・生成・ロードは現行 `Chunk` 粒度が適切。
- スコープ:
  - 地形更新トリガは `Sub-block` 単位へ移行し、地形台帳の正本と保存単位は `Chunk` のまま維持する。
- Subtasks:
  - [ ] 地形更新イベントを `Sub-block` dirty キューへ変換する。
  - [ ] 連結/破壊判定の起動範囲を `Sub-block + 境界マージン` へ局所化する。
  - [ ] `Sub-block` 結果を `Chunk` 台帳へ一括コミットする段を実装する。
  - [ ] `Chunk dirty` と `Sub-block dirty` の二層管理を実装する。
  - [ ] セーブ時は `Chunk` 差分のみを正本として書き出すことを検証する。
  - [ ] 単体テストを追加する（境界跨ぎ破壊、コミット順、dirty伝播）。
- 完了条件:
  - 地形更新は局所 `Sub-block` で起動しつつ、保存/再生成の整合性が `Chunk` 単位で維持される。

### [PHYS-05] XPBD外力統合によるマルチレート安定化

- Status: `Planned`
- 背景:
  - `Sub-block` マルチレートで更新頻度を下げると、重力などの外力を通常積分するだけでは再更新時の拘束補正が大きくなり、跳ねやすくなる。
  - 低活動ブロックを高レベル（低頻度）に維持したまま安定性を確保したい。
- スコープ:
  - 外力（まず重力）をXPBD拘束解法へ統合し、更新頻度が低いブロックでも拘束と整合した形で解く。
  - 第1段階は粉体（XPBD）を対象にし、流体（PBD）は後続タスクで扱う。
- Subtasks:
  - [ ] 粉体XPBDの拘束式で外力項（重力）を右辺へ取り込む実装方針を確定する。
  - [ ] `scheduled` 粒子のみ更新する現行マルチレート経路と、外力統合後の時刻扱い（有効dt）を整合させる。
  - [ ] 粉体の地形/粒子接触で、外力込みでも過補正しないようコンプライアンスと反復条件を調整する。
  - [ ] 静止近傍ケース（地面上の粉体）で高level維持時の安定性テストを追加する。
  - [ ] 既存シナリオ・単体テストで回帰確認する。
- 完了条件:
  - 粉体で `max_level` が高い状態でも、地面接触時の周期的な跳ねが抑制され、更新頻度低下の効果を維持できる。

  - [x] fBm評価を導入し、フットプリントに応じて高周波オクターブ寄与を減衰/打ち切りする。

### [REND-01] Tileベース描画パイプライン再編（Full/LOD統合）

- Status: `In Progress`
- 背景:
  - 現状はフルチャンク描画とLOD描画の管理単位が分かれており、同期コストと責務境界が不明瞭。
  - ズーム時に `sync_lod_chunks_to_render` などの更新コストが高く、Tile単位の並列更新へ寄せたい。
- スコープ:
  - Render側の最小単位を `Tile` に統一し、Full/LOD を「解像度違いの同一表現」として扱う。
  - LOD要求はカメラ情報のみで決定し、物理演算範囲（active/halo）とは独立に管理する。
  - `World -> Tile` 反映と `Tile -> Texture` 更新を分離し、Tile単位に並列実行可能な構成へ移行する。
- Subtasks:
  - [x] `RenderTile` コンポーネント（tile座標、`lod_level`、参照チャンク範囲、dirty状態）を定義する。
  - [x] カメラ状態から required tile set を算出するsystemを実装する（Full/LOD統一）。
  - [x] requiredとの差分で Tile entity を spawn/update/despawn する reconcile system を実装する。
  - [x] 地形/粒子の更新から Tile dirty を付与する `World -> Tile` 反映systemを実装する。
  - [x] Tileテクスチャ生成をTile単位ジョブ化し、`Assets<Image>` 反映系と分離する。
  - [x] チャンク改変時のLOD再サンプリング範囲を影響Tileのみに局所化する。
  - [x] カメラ範囲外のTile破棄にヒステリシスを導入し、entity churnを抑制する。
  - [x] Grid Overlay をTile基準で再構成し、Full/LOD輪郭を一貫表示する。
  - [x] Tracyで `required calc / reconcile / reflect / texture upload` の内訳計測を追加する。
  - [ ] 旧実装比でズーム時の描画更新コストを比較し、改善を確認する。
- 完了条件:
  - Full/LOD描画が `Tile` 実装へ統合され、ズーム時のレンダリング更新負荷が現行より有意に低下する。
  - 描画LOD決定がカメラ依存のみで完結し、物理演算範囲とは独立して動作する。

## Design Feedback (from Impl sessions)

- 並列切替仕様の明文化が必要:
  - `SimulationParallelSettings.enabled` は active region 有効時でも尊重し、強制逐次化しない。
  - 並列経路でも `active/halo` 判定を逐次経路と同等に適用する。
- LOD粒子描画仕様の明文化が必要:
  - 退避（deferred）粒子は、live粒子が消えた後も Tile 合成経路で可視化を維持する。
  - LOD粒子は `1px` 固定ではなく、粒子半径をタイル解像度へ投影した半径で描画する。
- Sub-blockマルチレート仕様の明文化が必要:
  - 境界インパルス負債の集計は現状逐次経路で適用し、`parallel_enabled` 時は `rate>1` ブロック混在で逐次フォールバックする。
- マルチレート外力統合仕様の明文化が必要:
  - `scheduled` 粒子のみ更新する方針と、外力（重力）をXPBD拘束解法へ取り込む方針の整合条件を定義する。
  - 粉体（XPBD先行）と流体（後続でXPBD化含む）の適用順と互換条件を定義する。
