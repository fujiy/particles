# 物理シミュレーション×工業 2Dサンドボックス Design Doc

## 1. 本ドキュメントの役割

- `docs/design.md` は「現在の目標仕様」のみを記載する。
- 変更履歴、移行手順、どの版で何を変えたかは記載しない。
- 実装順序、作業背景、変更の文脈は `docs/tasks.md` 側で管理する。

## 2. プロダクト目標

- 2D横視点のドット絵サンドボックスで、流体・粉体・破壊を扱う。
- 物理の厳密性よりも、ゲームとして破綻しない安定挙動と一貫性を優先する。
- 工業レイヤ（設備・搬送・貯蔵）と自然物レイヤ（地形・流体・粉体）を接続する。
- 計算負荷は局所高忠実度、遠方低忠実度で一定化する。

## 3. 単位系と空間定義

- SIベースを使用し、Bevy座標は `1.0 = 1m` とする。
- 地形セルサイズは `CELL_SIZE_M = 0.25`。
- チャンクサイズは `CHUNK_SIZE = 32`（1チャンク `8m x 8m`）。
- 描画基準は `PIXELS_PER_METER = 64`、1セルは16px。
- 座標は `global_cell` / `chunk_coord` / `local_cell` を明示的に分離する。

## 4. 世界表現

### 4.1 TerrainWorld

- `TerrainWorld` はチャンク疎管理の地形台帳とする。
- セルは少なくとも以下を保持する。
  - `Empty`
  - `Solid { material, hp }`
- 地形は通常 `Frozen` とし、静的境界として扱う。
- 地形が破壊・分離した場合、条件に応じてオブジェクト化する。

### 4.2 ParticleWorld

- 粒子実体は `ParticleWorld` に SoA で保持する。
- 最低限の属性。
  - `pos`, `prev_pos`, `vel`, `mass`, `material`, `flags`
- 近傍探索は均一グリッド（空間ハッシュ/配列バケット）を使う。
- 反復計算で使うワークバッファを明示的に分離する。

### 4.3 ObjectWorld

- 物体は粒子集合とメタデータで管理する。
- 最低限の属性。
  - `ObjectId`
  - `particle_indices`
  - `rest_local`
  - `mass_sum`
- 形状拘束パラメータ（剛性、反復数）
- 描画は `1オブジェクト = 1Entity` とする。

### 4.4 物理更新ブロック（Sub-block）

- 保存・生成・ロードの単位は `Chunk` を維持する。
- 物理更新スケジューリングは `Chunk` より小さい `Sub-block` 単位で管理する。
- `Sub-block` は以下の状態を持つ。
  - 更新レートクラス（`rate=1/2/4` など）
  - 昇格/降格ヒステリシス用カウンタ
  - 境界インパルス負債（後述）
- 地形破壊・連結評価・粒子拘束は `Sub-block` で起動するが、最終コミット先は `Chunk` 台帳とする。

## 5. マテリアルと形態

- 同じ材質でも形態を分けて扱う。
  - `SolidCell`
  - `GranularParticle`
  - `FluidParticle`
- 主要材質。
  - 液体: `Water`
  - 固体/粉体系: `Rock`, `Gravel`, `Sand`, `Soil`
- `PARTICLES_PER_CELL[material]` でセル→粒子分解密度を制御する。
- 固体→粉体（例: `Rock -> Gravel`）は破壊イベントで実行する。
- 粉体→固体逆変換は現時点で対象外。

## 6. 物理シミュレーション

### 6.1 基本フロー

- `FixedUpdate` で固定刻み更新する。
- 1サブステップの標準フロー。
  1. 予測積分
  2. 近傍グリッド更新
  3. 拘束反復
  4. 位置適用
  5. 速度更新
  6. 休眠判定

### 6.2 水（液体）

- 水は密度拘束（PBF/XPBD系）を使う。
- 地形との干渉はSDF法線に沿った `boundary_push` で処理する。
- 岩側のSPH寄与は使わない。
- `halo` 領域の水粒子は押し戻し専用にはせず、`lambda` 計算へ参加する圧力境界として扱う。
- 貫通抑制は `substeps` / 反復数 / 押し戻し係数 / 速度クランプで制御する。

### 6.3 粉体（非水接触）

- 粉体はXPBD接触拘束を使う。
- 必須拘束。
  - 法線非貫通
  - 接線摩擦（`mu_s`, `mu_k`）
- 反発係数 `e` は速度更新段で適用する。
- 粉体・地形・オブジェクトの接触は同一APIで扱い、相手の `inv_mass` で分岐する。

### 6.4 物体（ほぼ剛体）

- 形状マッチングを主軸に使う。
- `rest_local` をCOM基準で保持し、2D極分解で回転を解く。
- 形状変化がない間は剛体近似として安定挙動を優先する。

### 6.5 破壊

- 破壊トリガは次を併用する。
  - 衝突インパルス閾値
  - 変位/ひずみ閾値（持続荷重）
- 片方だけに依存せず、動的・準静的の両破壊モードを扱う。
- 破壊セルのみを除去対象とし、無関係セルの同時破壊は行わない。

### 6.6 分割と分離

- 連結判定は4近傍で行う。
- オブジェクトは破壊後に連結成分分解し、最大成分を既存IDとして維持する。
- 地形分離は上限付きflood fillで近似する。
  - `count <= DETACH_FLOOD_FILL_MAX_CELLS` は分離塊としてオブジェクト化
  - `count > DETACH_FLOOD_FILL_MAX_CELLS` は地形として保持

### 6.7 Sleep / Wake

- 粒子は `Active` / `Sleeping` を持つ。
- Sleep条件は変位・速度・継続フレームで判定する。
- Wake条件は接触補正、衝突インパルス、編集イベントで判定する。
- Wakeは近傍伝播し、閾値ヒステリシスで振動を抑える。
- 破壊直後に生成された粉体は一定フレームSleep禁止にする。

### 6.8 物理演算領域（active / halo / inactive）

- 物理演算領域は `active / halo / inactive` の3層で管理する。
- `active` はカメラ中心チャンクから `active_radius_chunks` 以内を基準に構築し、遠方の一時的な `Active` 粒子だけで領域を肥大させない。
- `halo` は位置更新を行わない凍結境界層だが、速度属性は保持する。
- 境界バッファ投入は「`active -> halo` へ越境した瞬間」のみ実行する。
- `active` 縮小で結果的に `halo` 化した粒子は境界バッファへ再投入しない。
- `active + halo` は常駐対象とし、退避キュー（far-field）は `live/freeze` 外の `inactive` のみを対象にする。

### 6.9 流体マルチレート更新（固定dt）

- 粒子ごとの可変dtは採用しない。`fixed_dt` は全体で共通とする。
- 計算負荷削減は `Sub-block` 単位の更新頻度で実現する（例: `rate=1/2/4`、最大更新比は `1:4`）。
- `rate>1` のブロックは、非更新フレームで拘束解法を省略し、必要最小限の予測/状態保持のみ行う。
- 更新レートが異なるブロック境界では、以下を適用する。
  - 高頻度側は通常どおり拘束を解く。
  - 低頻度側への反作用は「境界インパルス負債」として蓄積する。
  - 低頻度側の更新フレームで負債を適用してから拘束解法を実行する。
- 境界インパルス負債が上限を超える場合は、対象 `Sub-block` を強制昇格する。
- `active` 側から低頻度ブロックへ粒子侵入が生じた場合、侵入先を即時昇格する（近傍伝播あり）。
- 近距離領域では far-field 境界バッファを使わず、上記昇格ルールを優先する。

## 7. 相互作用

### 7.1 流体・粉体と地形

- 流体はSDF境界で押し戻しを受ける。
- 粉体はXPBD接触拘束で地形と干渉する。
- 地形は通常 `Frozen` だが、破壊判定に必要な局所評価は行う。

### 7.2 流体・粉体とオブジェクト

- オブジェクトはローカルSDFを持つ。
- 物理グリッドは候補 `ObjectId` を持ち、`distance/normal` はローカルSDF直接評価で得る。
- 反作用はオブジェクト単位で集計する。

## 8. レンダリング

### 8.1 地形

- チャンクごとに `Image + Sprite` で描画する。
- dirtyチャンクのみ再ラスタライズする。

### 8.2 流体・粉体

- 世界側密度テクスチャに投影してドット描画する。
- 水は閾値処理と平滑化で液体境界を形成する。
- 岩セル上の水ドットは抑制し、地形表示を優先する。

### 8.3 オブジェクト

- オブジェクトはローカル座標基準のドットパターンを持つ。
- 移動・回転時は見た目を追従させる。
- 形状不変時は `Transform` 更新のみ、形状変更時のみ再ラスタライズする。

### 8.4 デバッグ表示

- 粒子オーバーレイ、グリッドオーバーレイ、HUD（FPS等）を切替可能にする。
- 粒子オーバーレイでは `halo` 粒子を表示対象に含める。
- 地形粒子のオーバーレイ表示は「物理演算範囲 + halo」内に限定する。
- `Sub-block` ごとの更新レートクラス可視化を提供する。
- 境界インパルス負債のヒートマップ可視化を提供する。

### 8.5 Tileベース描画とLOD統一

- `Chunk` は物理演算単位、`Tile` は描画単位として責務を分離する。
- Full/LOD は別機構ではなく、`lod_level` が異なる `Tile` として統一管理する。
- required tile set はカメラ位置とズームのみから決定し、`active/halo` 状態には依存させない。
- カメラ近傍ほど高解像度 `Tile` を要求し、未充足時は要求解像度で再生成する。
- Tileの状態は `Required -> Loaded -> Dirty -> Uploaded -> Evictable` で管理する。
- Tile破棄にはヒステリシスを設け、spawn/despawnスパイクを抑える。

## 9. ECSとモジュール方針

- 主要責務はBevy pluginで分離する。
  - Physics
  - Render
  - Interface
  - Overlay
  - Camera
- 粒子はEntityで持たず、`Resource` として一括管理する。
- Entity側は必要なハンドルと表示状態のみ持つ。

## 10. 並列化とGPU移行方針

- 物理は1つの主要system内でフェーズ分解し、自前並列化する。
- データレイアウトはSoA + 連続メモリ近傍グリッド（`cell_start/end`）を維持する。
- 共有加算はスレッドローカル集計後にreduceする。
- CPU最適化時も、後でcompute shaderへ置換可能なI/O境界を維持する。
- 近傍探索グリッドは密配列確保が上限を超える場合、sparse表現へフォールバックする安全策を持つ。
- Renderでは `World -> Tile` 反映と `Tile -> Texture` 生成をTile単位の並列化対象とする。
- `Assets<Image>` 反映は競合回避のため専用段に分離し、計測で律速点を確認する。
- マルチレート実装では `Sub-block` ごとのジョブ実行順を決定論的に固定し、境界負債集計はスレッドローカル集計後reduceする。

## 11. セーブ/ロードと世界生成

### 11.1 セーブ/ロード

- 保存対象。
  - 地形
  - 粒子
  - オブジェクト
  - 互換判定に必要なメタ情報
- `save_version` と互換チェックを必須にする。
- ロード時は整合性検証（ID重複、参照不整合、境界外）を行う。
- `Sub-block` の更新レート状態はランタイム最適化情報として扱い、セーブの正本対象には含めない（ロード後に再推定）。

### 11.2 決定論的地形生成

- ワールドはseedベースの決定論生成を前提とする。
- 生成関数は座標とseedから同じ結果を返す純関数として設計する。
- 変更が入ったチャンクのみ保存し、未変更チャンクは再生成する。
- 地形高低差の一次生成は OpenSimplex ノイズで行う。
- 基本ルールは以下とする。
  - 各 `x` に対して OpenSimplex を評価し、`surface_y(x)` を決定する。
  - `y > surface_y(x)` は `Empty`。
  - `y <= surface_y(x) - SOIL_DEPTH_CELLS` は `Rock`。
  - `surface_y(x) - SOIL_DEPTH_CELLS < y <= surface_y(x)` は `Soil`。
- 初期フェーズでは地形生成の対象を「地表高低差 + 表土層」のみに限定し、水域・洞窟・鉱石・植生は生成しない。
- `surface_y(x)` はチャンク境界に依存せず、ワールド座標で直接評価する。

### 11.3 LODサンプリング

- 遠景表示や超拡大表示のため、任意解像度の地形サムネイル生成を可能にする。
- フル解像度チャンクを生成せず、LODセル単位の直接サンプリングで近似する。
- 連続LOD生成は fBm を基本とし、フットプリント依存で有効オクターブ寄与を制御する。
- ベースノイズは差し替え可能とし、必要に応じて複数ノイズ合成を許容する。
- 材質は `Stone/Soil/Empty` の確率場で評価し、確率和は1.0へ正規化する。
- 深度依存の遷移関数で確率場を連続化し、改変済みセル/チャンクは確率場より優先する。
- 近景は決定的サンプリング、遠景は期待値色を使い、中間帯は連続補間する。
- 遠景でフレームごとのランダム抽選は行わず、時間方向の安定性（ちらつき防止）を優先する。
- 期待値色への遷移は固定距離ではなく `stderr / sqrt(N_eff)` ベースで決定する。
- `N_eff` 近似に使う相関長（cells）は調整可能パラメータとする。
- `span_chunks <= CELL_PIXEL_SIZE` の近景LODでは期待値ブレンドを使わず、ドットサンプリングを維持する。

## 12. テスト方針

### 12.1 Unit test

- 単体ロジックは実装近傍で `#[cfg(test)]` により検証する。

### 12.2 Physics integration test

- headlessシナリオテストを `tests/` で実行する。
- `ScenarioSpec` を使い、固定dt・固定stepで再現性を確保する。
- 終了状態は artifact として出力する。
  - `final_state.json`
  - `metrics.json`
  - 必要に応じて `final_state.png`

### 12.3 簡易再生モード

- シナリオを簡易UIで再生・停止・step実行・ループ再生できる。
- テスト実行と同じシナリオ定義を共有し、結果比較を容易にする。
