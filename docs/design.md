# 物理シミュレーション×工業 2Dサンドボックス Design Doc

## 1. 本ドキュメントの役割

- `docs/design.md` は現在の目標仕様のみを記載する。
- 実装順序、背景、移行文脈は `docs/tasks.md` で管理する。

## 2. プロダクト目標

- 2D横視点のドット絵サンドボックスで、流体・地形・工業設備の相互作用を扱う。
- 連続体の中核ソルバは MLS-MPM（明示更新）とし、広域で安定した計算資源配分を実現する。
- 近傍高忠実度、遠方低忠実度（空間 LoD + 時間 LoD）を一貫した仕様として扱う。
- 保存量は質量・運動量を優先し、エネルギーは数値散逸を許容しつつ観測可能にする。

## 3. 単位系と空間定義

- SIベースを使用し、Bevy座標は `1.0 = 1m` とする。
- 地形セルサイズは `CELL_SIZE_M = 0.25`。
- チャンクサイズは `CHUNK_SIZE = 32`（1チャンク `8m x 8m`）。
- 描画基準は `PIXELS_PER_METER = 64`、1セルは16px。
- 座標系は `global_cell` / `chunk_coord` / `local_cell` / `world_pos_m` を明示的に分離する。

## 4. 世界表現

### 4.1 TerrainWorld

- `TerrainWorld` はチャンク疎管理の地形台帳とする。
- セルは少なくとも以下を保持する。
  - `Empty`
  - `Solid { material, hp }`
- 地形は通常 `Frozen` の静的境界として扱い、MLS-MPM 側にはSDF境界として供給する。

### 4.2 ContinuumParticleWorld

- 連続体粒子は SoA で保持する。
- 水粒子の最低限属性。
  - `x_p`（位置）
  - `v_p`（速度）
  - `m_p`（粒子質量）
  - `V0_p`（初期体積）
  - `F_p`（変形勾配）
  - `C_p`（APIC affine 行列）
  - `material_id`
  - `owner_block_id`（現在の所有 block）
- 水優先フェーズでは `material_id=Water` のみを有効化する。
- 粒子SoA本体は単一 `Resource` とし、blockごとの粒子配列複製は行わない。

### 4.3 GridHierarchy

- MLS-MPM 計算格子は block 化した階層グリッドで管理する。
- block は以下を持つ。
  - 格子ピッチ `h_b`
  - アクティブ格子ノード集合
  - level index（空間LoD）
  - subcycling 用の時間刻み `dt_b`
- 格子ノードは以下を保持する。
  - `m_i`（ノード質量）
  - `p_i`（ノード運動量）
  - `v_i`（ノード速度）
  - `f_i`（内部力）

### 4.4 CouplingWorld

- 連成層は「境界条件と作用反作用の交換」を責務に持つ。
- 水優先フェーズの連成対象。
  - 地形SDF境界（必須）
  - 剛体/粉体連成はインターフェース定義のみ保持し、計算本体は有効化しない。

### 4.5 物理更新ブロック（Sub-block）

- 保存・生成・ロードの正本単位は `Chunk` を維持する。
- 物理更新スケジューリングは `Sub-block` 単位で管理する。
- `Sub-block` は以下を持つ。
  - `level`（空間LoDレベル）
  - `dt_b`（ローカル時間刻み）
  - `activity`（速度・発散・境界接触の統計）
  - `promotion/demotion` ヒステリシス状態
  - `owner_indices`（更新責務を持つ粒子index集合）
  - `ghost_indices`（近傍参照のみ行う粒子index集合）
- `owner_indices` は block 間で互いに素であることを不変条件とする。

### 4.6 Block Index Table

- blockごとの index table（`owner_indices` / `ghost_indices`）は `Resource` で管理する。
- index table は毎substepで全再構築せず、粒子の block 跨ぎ移動または領域更新時のみ再binningする。
- `ghost_indices` は read-only 参照用途に限定し、G2Pで直接更新しない。

## 5. マテリアル

- 粒子は `material_id` を持つ。
- 物性は `material_id -> category -> parameter set` の参照で与える。
- 水優先フェーズで必須な物性。
  - `rho0`（基準密度）
  - `bulk_modulus` または `eos_stiffness`
  - `gamma`（EOS指数）
  - `mu`（粘性係数）

## 6. 物理シミュレーション（MLS-MPM）

### 6.1 基本フロー

- `FixedUpdate` で外側の基準刻み `dt_frame` を進める。
- 各 frame で block ごとに subcycling を行い、1サイクルで以下を実行する。
  1. `P2G`: 粒子質量・運動量・応力寄与を格子へ転送
  2. `Grid Update`: 外力・内部力・境界条件を適用してノード速度更新
  3. `G2P`: 格子速度から粒子速度・位置・`C_p`・`F_p` を更新
  4. `Post Process`: 体積変化率や監視量を集計

### 6.2 水の構成則

- 水は弱圧縮性モデルを採用する。
- 圧力は体積比 `J_p = det(F_p)` または密度比で評価する。
- 最低限の仕様。
  - 圧力項は等方応力として扱う。
  - 粘性は速度勾配由来の散逸項として扱う。
  - 張力モデルは初期フェーズでは導入しない。

### 6.3 境界条件（地形）

- 地形境界はSDFで与える。
- 格子ノードまたは粒子予測位置が `phi(x) < 0` の場合に補正する。
- 補正仕様。
  - 法線方向の非貫通を強制する。
  - 接線は摩擦係数に応じて減衰する。
  - 境界補正は作用反作用ログに記録可能な形式で保持する。

### 6.4 時間刻みと安定条件

- block 時間刻みは次式で決める。
  - `dt_b = min(C_u * h_b / u_max_b, C_c * h_b / (c_b + u_max_b), C_a * sqrt(h_b / a_max_b))`
- `u_max_b` は block 内最大速度、`c_b` は見かけ音速、`a_max_b` は最大加速度。
- safety factor `C_u, C_c, C_a` は 1 未満の定数とする。

### 6.5 空間LoD + 時間LoD

- 空間LoD。
  - 高活動領域は細かい `h_b`。
  - 低活動・遠方は粗い `h_b`。
- 時間LoD。
  - 各 block は `h_b` と局所速度場から `dt_b` を持つ。
  - 粗い block は更新頻度を下げる。
- LoD 境界では質量・運動量フラックス交換を明示し、保存則を優先する。

### 6.6 物理演算領域（active / halo / inactive）

- `active`: 粒子更新と格子更新を実行する領域。
- `halo`: active 境界のフラックス整合に必要な補助領域。
- `inactive`: 計算を停止し、統計情報のみ保持する領域。
- 領域判定はカメラ依存ではなく、物理活動量と相互作用要求で決める。

### 6.7 保存量ポリシー

- 質量保存は P2G/G2P の離散化で満たす。
- 運動量交換（境界・連成・LoD境界）は必ず対称更新する。
- エネルギーは厳密保存を要求しないが、散逸量の可視化メトリクスを保持する。

### 6.8 block単位並列実行（CPU先行）

- `P2G -> Grid Update -> G2P` はフェーズ単位で実行し、各フェーズ内で block 並列化する。
- `P2G` は owner/ghost 参照を許可するが、書き込み先は当該 block のノード配列に限定する。
- `G2P` は `owner_indices` のみ更新し、`ghost_indices` は更新しない。
- 粒子可変アクセスは「ownerの非重複」を前提とし、デバッグビルドで owner 一致と重複なしを検証する。
- 高速経路で `unsafe` を使う場合でも、上記不変条件を満たす検証段を必須とする。

### 6.9 段階導入（空間幅均一 -> 空間LoD）

- 第1段階は空間幅均一（全blockで同一 `h_b`）で block 並列 + 時間LoD を導入する。
- 第2段階で空間LoD（blockごとに `h_b` 可変）を導入し、粗密境界のフラックス交換を追加する。
- 第1段階では粒子の merge/split（再サンプル）は導入しない。

## 7. 連成ポリシー

### 7.1 地形連成（有効）

- 地形は静的境界として MLS-MPM に接続する。
- 破壊や地形改変は `TerrainWorld` の責務とし、物理フレーム境界でSDFを更新する。

### 7.2 剛体・粉体連成（定義のみ）

- インターフェースは定義するが、水優先フェーズでは計算を有効化しない。
- 連成時に満たすべき条件。
  - 接触での作用反作用の対称記録
  - 運動量交換の保存
  - 連成順序の決定論性

## 8. レンダリング

### 8.1 地形

- チャンク単位で `Image + Sprite` として描画する。
- dirtyチャンクのみ再ラスタライズする。

### 8.2 水

- 水粒子はタイル合成経路へ投影して描画する。
- 近景は粒子寄与を高く、遠景は密度場ベースで平滑に表示する。

### 8.3 Tileベース描画とLOD

- `Chunk` は物理単位、`Tile` は描画単位として分離する。
- Full/LOD は `lod_level` の違いとして統一管理する。
- required tile set はカメラ位置・ズームのみで決定する。

## 9. ECSとモジュール方針

- 主要責務は plugin で分離する。
  - `physics_mpm`
  - `terrain`
  - `render_tile`
  - `interface`
  - `overlay`
- 連続体粒子は `Resource` 一括管理を基本とし、粒子Entity化は行わない。
- blockの計算本体データ（grid配列、index table）は `Resource` 管理を基本とする。
- blockをEntity化する場合は可視化/統計などの軽量メタデータに限定し、数値カーネルは `Resource` 経由で実行する。

## 10. セーブ/ロードと世界生成

### 10.1 セーブ/ロード

- 保存対象。
  - 地形差分
  - 連続体粒子状態
  - 互換判定メタ情報（`save_version`, `generator_version`）
- `Sub-block` のスケジューラ状態はランタイム最適化情報として扱い、保存正本には含めない。

### 10.2 決定論的地形生成

- ワールドは seed ベースの決定論生成を前提とする。
- 未変更チャンクは再生成し、変更チャンクのみ差分保存する。
- 地形のLODサンプリングは連続関数評価を許容し、近景は決定的サンプリングを維持する。

## 11. テスト方針

### 11.1 Unit test

- 転送カーネル、構成則、CFL判定、境界補正を単体テストで検証する。

### 11.2 Physics integration test

- headless シナリオテストで固定入力・固定stepの再現性を検証する。
- 水優先フェーズの必須シナリオ。
  - 静水保持
  - 柱崩壊（dam-break）
  - 地形接触（侵入率上限）
  - LoD境界通過での質量保存

### 11.3 計測

- フレームごとに以下を収集する。
  - 総質量誤差
  - 総運動量誤差
  - 最大CFL比
  - LoD境界フラックス残差
